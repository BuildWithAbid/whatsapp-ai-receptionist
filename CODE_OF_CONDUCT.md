# Code of Conduct

## Our Pledge

We want this project to be useful to founders, developers, agencies, and local businesses building automation responsibly. To support that, we are committed to making participation in this project a harassment-free experience for everyone.

## Our Standards

Examples of behavior that contribute to a positive environment:

- Being respectful, direct, and constructive
- Assuming good intent while still holding a high technical bar
- Welcoming questions from contributors with different levels of experience
- Giving actionable feedback on code, docs, UX, architecture, and operations
- Sharing context when disagreeing instead of dismissing others

Examples of unacceptable behavior:

- Harassment, discrimination, or abusive language
- Personal attacks, insults, or bad-faith argumentation
- Publishing private information without explicit permission
- Spam, trolling, or intentionally derailing technical discussions
- Requests or contributions that try to add deceptive, illegal, or unsafe automation behavior

## Enforcement Responsibilities

Project maintainers are responsible for clarifying and enforcing this code of conduct. They may remove, edit, or reject comments, commits, code, wiki edits, issues, and other contributions that violate these standards.

## Scope

This Code of Conduct applies within project spaces and in public spaces when an individual is representing the project or its community.

## Enforcement

Report unacceptable behavior by opening a private security-style disclosure if appropriate or by contacting the maintainer through the repository owner profile.

Maintainers may take any action they deem appropriate, including warning, temporary restriction, or permanent removal from the project community.
